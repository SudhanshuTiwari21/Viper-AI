## Before you begin

You’ll need:

- A **project folder** on your machine (your app, library, or monorepo root).
- A **network connection** for AI features that call hosted models.
- When accounts are enabled for your environment, a **Viper sign-in**—same email flow you’d expect from any modern product (no passwords to memorize in the client).

Public installers and managed hosting are **rolling out with early access**. Until your build is available, teammates who clone the repo can run the desktop app locally—see [Build from source](/docs/guide/for-developers).

## Open Viper and choose a workspace

1. Launch **Viper Desktop**.
2. **Open folder** and pick the repository or directory you want to work in. That folder is your workspace; the AI uses it as context for chat and actions.
3. Open the **Chat** panel when you’re ready to talk to the model.

## First message

Try something small and specific, for example:

- “Summarize what this package does and list the main entry points.”
- “What’s the safest way to add error handling around this API call?”

If the reply fails or hangs, see [Troubleshooting](/docs/guide/troubleshooting).
