title: Chat
description: Pick a mode that matches how you want to work—question, plan, debug, or delegate.
---

The chat sits beside your editor so you can **iterate without tab-hopping**. Use the mode menu to steer tone and depth; you can change it anytime in a thread.

## Ask

Best for **quick questions** and explanations: “What does this file do?”, “How do I run tests?”, “Explain this error.” The model answers from your workspace context when it helps.

## Plan

Best when you need **structure before code**: milestones, tradeoffs, checklists, or a phased approach. Use Plan when you’d otherwise open a doc or whiteboard.

## Debug

Best when something **isn’t working** and you want a focused investigation: stack traces, failing tests, or “why does this path never run?” Pair it with the files or logs you care about.

## Agent

Best when you want the model to **propose or apply multi-step changes** across files—refactors, scaffolding, repetitive edits—while you stay in the loop to review.

## Tips

- **Name the outcome** (“I want X behavior by Friday”) so the model can align suggestions.
- **Switch modes** if the first reply is too shallow (Ask) or too heavy (Agent).
- If streaming stops mid-reply, see [Troubleshooting](/docs/guide/troubleshooting).
