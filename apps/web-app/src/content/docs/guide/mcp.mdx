Viper supports the **Model Context Protocol (MCP)** — an open standard for connecting AI agents to development environments. With MCP, you can control Viper agents directly from your editor.

---

## What is MCP?

MCP is a protocol that allows language models and AI agents to interact with tools, files, and external services through a standardized interface. Viper implements MCP to expose its agent fleet to any MCP-compatible client.

## Supported Clients

- **Cursor** — Native MCP support via settings
- **VS Code** — Via the Viper VS Code extension
- **Claude Desktop** — Add Viper as an MCP server

## Setup in Cursor

Add the following to your Cursor `settings.json`:

```json
{
  "mcp.servers": {
    "viper": {
      "command": "viper",
      "args": ["mcp", "start"],
      "env": {
        "VIPER_WORKSPACE": "my-project"
      }
    }
  }
}
```

## Setup in VS Code

Install the Viper extension from the VS Code marketplace, then run:

```bash
viper mcp start --client vscode
```

## Available MCP Tools

Once connected, the following tools are exposed to your editor:

| Tool | Description |
|------|-------------|
| `viper.run` | Run a prompt against the full agent fleet |
| `viper.context` | Query the shared context graph |
| `viper.status` | Check agent and pipeline status |
| `viper.rollback` | Undo the last agent action |
